Overview

What This Platform Does

The Challenge

Enterprise security teams often have thousands of findings across identity, network, data protection, logging, detection, and compliance services. The challenge is not just seeing the findings — it is knowing which actions matter most, what they impact, how to remediate safely, and how to prove improvement over time.

The Platform

The Agentic GenAI Security Maturity Accelerator centralizes AWS security findings, scores maturity across key pillars, explains risk in plain language, investigates live AWS context using read-only tools, generates remediation plans, routes actions through approval, supports dry-run execution, and enables controlled low-risk remediation with full auditability.

Security Findings Pipeline

Imports AWS security findings, normalizes them into a consistent format, removes duplicate noise, and maps issues to maturity pillars and remediation actions.

Maturity Scoring Engine

Calculates a 0–5 maturity score across security pillars and identifies the controls that have the greatest impact on improving the customer's posture.

Agentic AI Control Plane

Uses Amazon Bedrock and governed tool registration to explain findings, investigate AWS context, generate remediation plans, and guide approved workflows without giving the model unrestricted AWS access.

Controlled Remediation Workflow

Supports queueing, approval, dry-run execution, before/after state capture, verification, audit logging, and limited low-risk live remediation through an allowlisted execution path.

Capabilities

Security Maturity Capabilities

Security Maturity Scoring

Calculates a 0–5 maturity score across identity, network security, data protection, detection, vulnerability management, and operational readiness.

Findings Deduplication

Groups repeated findings by control and resource count so teams can focus on root causes instead of duplicate noise.

AWS Documentation Grounding

Links remediation guidance to official AWS documentation and clearly labels response sources so recommendations remain explainable.

Read-Only AWS Investigation

Uses read-only AWS tools to inspect IAM, S3, EC2 security groups, CloudTrail, Security Hub, GuardDuty, and Inspector context when investigation mode is enabled.

Remediation Planning

Generates step-by-step remediation plans, AWS CLI examples, Terraform patches, CloudFormation examples, rollback plans, blast-radius analysis, and estimated score impact.

Remediation Queue and Approval

Routes proposed actions into a queue with unique remediation action IDs, risk labels, approval status, and clear review context before execution.

Dry-Run Execution

Runs the full execution workflow safely with simulated before-state, after-state, verification, and audit logging without changing AWS resources.

Low-Risk Live Remediation

Supports approved low-risk remediation only when explicitly enabled, using an execution role, allowlist checks, before/after capture, verification, and audit logging.

Audit and Governance

Records queued, approved, rejected, skipped, dry-run, blocked, and live execution events to support traceability and security governance.

Governance

Built for Safe Enterprise Remediation

Feature-flag controlled capabilities Read-only investigation mode Dry-run execution by default Human approval required Unique remediation action IDs Allowlisted low-risk live actions only Medium/high-risk live remediation disabled by default Blocklisted destructive actions Separate read-only and execution roles Before-state and after-state capture Post-execution verification Append-only audit logging AWS documentation grounding No remediation from chat text alone

Enterprise safety model: The platform is designed to increase security maturity without giving AI unrestricted control. Bedrock provides reasoning, but the backend control plane decides which tools are available, which IAM role is used, whether approval exists, whether dry-run or live mode is active, and whether an action is safe to execute.

Operations

Operational Workflow

1

Connect AWS Account

Connect to the customer AWS account using configured credentials or roles.

2

Run Security Assessment

Run Prowler or import AWS security findings into the findings pipeline.

3

Normalize and Score

Normalize findings, deduplicate repeated issues, map findings to pillars, and calculate a 0–5 maturity score.

4

Investigate With Read-Only Tools

Use approved read-only AWS tools to inspect identity, storage, network, logging, detection, and vulnerability context.

5

Generate Remediation Plan

Create remediation steps, risk classification, blast-radius analysis, CLI/IaC examples, rollback plan, and estimated score impact.

6

Queue and Approve

Add the action to the remediation queue and approve or reject it through the dashboard approval modal.

7

Execute Safely

Run dry-run execution by default. If explicitly enabled, execute only approved low-risk allowlisted actions using the execution role.

8

Verify and Audit

Capture before-state, after-state, verification result, and audit log evidence for the remediation lifecycle.
